À proposdes gestionnaires de paquetsde Node.js

  • Un gestionnaire de paquets permet de télécharger et de gérer les dépendances d'un projet.
  • Il en existe deux : Yarn et NPM

NPM

  • Package Manager par défaut de Node.js
  • Il est lié à la version de Node.js
  • Toute l'équipe doit utiliser la même version de Node.js, et donc de NPM, afin d'éviter les conflits du fichier de verrouillage des versions des paquets.

Yarn

  • Un peu plus rapide que NPM
  • Il n'est pas lié à la version de Node.js
  • Il est possible d'installer Yarn en utilisant NPM mais ça n'est pas recommandé.
  • En plus de devoir utiliser la même version de Node.js, toute l'équipe doit utiliser la même version de Yarn afin d'éviter les conflits du fichier de verrouillage des versions des paquets.
