InstallerNode.js

Les gestionnairesde version

Pour passer rapidement et proprement d'une version à l'autre de Node.js, il est plus que conseillé d'utiliser un gestionnaire de version :

  • nvm-windows pour windows
  • nvm pour tout le reste

Node.js est livré avec NPM, son gestionnaire de paquets par défaut. Il est important que tout les membres d'une équipe utilisent la même version de Node.js et de NPM afin d'éviter de sérieux conflits de version de dépendances.

Peut importe l'OS, Node.js est difficile à désinstaller proprement. Afin de désinstaller une version qui n'aurait pas été installée par un gestionnaire de version, veuillez vous référer aux documentations correspondantes aux versions de Node.js et de votre OS.

À propos des versions deNode.js

Il existe deux types de versions : les standards et les LTS (Long Term Support)

La version la plus récente est la 12.3.1 (le 29/05/2019). Il s'agit d'une version standard. C'est à dire qu'il n'y aura plus aucun support dès que la version 13 sortira.

La version LTS la plus récente est la 10.16.0 (le 29/05/2019). Elle date de 2018 et de nouvelles mises à jour de sécurité sortiront jusqu'en avril 2021.

Installer Node.js avecnvm-windows

Un installeur existe pour nvm-windows. Il est disponible sur son dépôt Github. Téléchargez puis installez le.

Une fois installé, depuis la ligne de commande : 

# installer la version 10.16.0 sur votre système
nvm install 10.16.0

# Lister les versions installées (et donc utilisables) sur votre système
nvm list

# utiliser la version 10.16.0
nvm use 10.16.0

# afin de vérifier quelle est la version effectivement utilisée
node -v

Ne pas oublier de prendre le temps de lire la documentation de nvm-windows !

Installer Node.js avecnvm

Utiliser le script d'installation via curl ou wget 

# avec curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.34.0/install.sh | bash

# avec wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.34.0/install.sh | bash

Télécharger et utiliser la version 10.16.0 

nvm install 10.16.0
nvm use 10.16.0

# faire de la version 10.16.0 la version par défaut de votre système
nvm alias default 10.16.0

Ne pas oublier de prendre le temps de lire la documentation de nvm, disponible sur son dépôt Github !
