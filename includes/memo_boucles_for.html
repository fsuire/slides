Petit mémo surles boucles for

À l'ancienne

"use strict";

const unTableau = [1, 2];
let sum = 0;

for (let i = 0 ; i < unTableau.length ; i++) {
  sum += unTableau[i];
}

console.log(sum); // 3

Les boucles for ne fonctionnent pas avec des tableaux indexés !

Ce type de boucle for permet l'utilisation de break et continue.

for in & for of

"use strict";

const unTableau = [1, 2];

let sum = 0;
for (let i in unTableau) {
  sum += unTableau[i];
}
console.log(sum); // 3

sum = 0;
for (let i of unTableau) {
  sum += i;
}
console.log(sum); // 3

Les boucles for ne fonctionnent pas avec des tableaux indexés !

Ce type de boucle for permet l'utilisation de break et continue.

.forEach

"use strict";

const unTableau = [1, 2];

let sum = 0;
unTableau.forEach((value, index) => sum += value);

console.log(sum); // 3

Les boucles for ne fonctionnent pas avec des tableaux indexés !

Quand il le peut, le javascripteux moderne préfère utiliser .forEach (et ses copines .map et .reduce).

Ce type de boucle ne permet pas l'utilisation de break et continue.

Comment itérer surun tableau indexé

"use strict";

const unTableauIndexé = { a: 1, b: 2 };
let sum = 0;

const keys = Object.keys(unTableauIndexé);

for (let key of keys) {
  sum += unTableauIndexé[key];
}
console.log(sum); // 3

sum = 0;
keys.forEach(key => sum += unTableauIndexé[key]);
console.log(sum); // 3

Oui, c'est moche ; mais rien n'empêche de créer sa propre fonction d'itération ou d'en utiliser une déjà présente dans des bibliothèques comme lodash.
