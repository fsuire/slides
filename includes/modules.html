Les modules

Notre code est fonctionnel, mais pas très beau. Nous ranger la fonction display dans un fichier src/utils.js.

Créer et remplir le fichier src/utils.js :

"use strict";
    
    function display(board) {
      // ...
    }
    
    module.exports = {
      display: display
    };

Importer la fonction display dans le fichier src/index.js :

// ...
    const { display } = require('./utils');
    // ...

Différents type d'importspour différents types de modules

"use strict";
    
    // on commence toujours par importer les modules de Node.js
    const { EOL } = require('os');
    
    // On importe ensuite les modules externes (provenant de NPM ou de github).
    // Comme il n'y a aucune différences entre les deux syntaxes,
    // il est de bon ton de sauter une ligne
    const figlet = require('figlet');
    const { Game } = require('morpion');
    
    // Viennent enfin les imports des modules locaux.
    // Bien que la syntaxe soit légèrement différentes
    // il est quand même de bon ton de sauter une ligne
    const { display } = require('./utils'); // Vous avez vu la différence syntaxique ?
                                            // on utilise un chemin relatif !

Sucre syntaxique pour tableaux indexés

Ces deux manières de faire sont complètement équivalentes :

const a = 'a';
    const b = 'b';
    
    const all = {
      a: a,
      b: b
    };
    
    module.exports = all;
const a = 'a';
    const b = 'b';
    
    // pas besoin de la variable intermédiaire `all`
    module.exports = {
      a, // pas besoin d'écrire `a: a`, car le nom
         // de l'index correspond au nom
         // d'une variable existante dans
         // le scope courrant
      b
    };
