Les points d'arrêts

Node.js permet l'utilisation de points d'arrêts. il suffit d'utiliser l'argument inspect lors du lancement du script.

node inspect src/index.js
nodemon inspect scr/index.js

Le mot-clef debugger; inséré n'importe où dans le code génère un point d'arrêt.

Lorsque l'exécution s'est arrêtée sur un point, plusieurs command sont possibles :

  • cont ou c : continue l'exécution en ignorant les points d'arrêts suivants
  • next ou n : continue l'exécution jusqu'au prochain point d'arrêt
  • step ou s : step in
  • out ou o : step out
